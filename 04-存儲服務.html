存儲服務

存儲服務是 AWS 的基礎服務之一，SAP 考試中涉及 S3、EBS、EFS、Storage Gateway 等關鍵知識點，以及如何設計高效、安全、經濟的存儲架構。

Amazon S3 (簡單存儲服務)

基本概念

  • 對象存儲服務
    • 高度可擴展、高可用性、高耐用性
    • 存儲無限量的數據
    • 適合非結構化數據
    • 支持任何類型的文件
  • 存儲桶和對象
    • 存儲桶：存儲對象的容器，全球唯一名稱
    • 對象：存儲在存儲桶中的文件和元數據
    • 對象大小：最大 5TB
    • 對象鍵：唯一標識存儲桶中對象的名稱
  • 數據一致性模型
    • 所有區域的所有操作都是強一致性
    • 新對象的 PUT 請求後立即可讀
    • 覆蓋和刪除現有對象的請求後立即可見

存儲類別

  • S3 標準
    • 默認存儲類別，高可用性和耐用性
    • 適合頻繁訪問的數據
    • 低延遲、高吞吐量
    • 支持多可用區域冗餘
  • S3 智能分層
    • 自動優化成本
    • 根據訪問模式在標準和不頻繁訪問層之間移動對象
    • 無性能影響
    • 無檢索費用
  • S3 標準 - 不頻繁訪問 (S3 Standard-IA)
    • 適合不經常訪問但需要快速訪問的數據
    • 比標準存儲成本低，但有檢索費用
    • 適合備份、長期存儲
    • 最低存儲持續時間：30 天
  • S3 單區域 - 不頻繁訪問 (S3 One Zone-IA)
    • 數據存儲在單個可用區
    • 比 S3 Standard-IA 成本低 20%
    • 適合可重新創建的數據
    • 最低存儲持續時間：30 天
  • S3 Glacier 即時檢索
    • 適合每季度訪問一次的歸檔數據
    • 毫秒級檢索時間
    • 比 S3 Standard-IA 成本低
    • 最低存儲持續時間：90 天
  • S3 Glacier 靈活檢索
    • 適合長期歸檔，偶爾需要檢索
    • 檢索時間：1 分鐘到 12 小時
    • 比即時檢索成本低
    • 最低存儲持續時間：90 天
  • S3 Glacier 深度歸檔
    • 最低成本存儲類別
    • 適合很少訪問的長期保存
    • 檢索時間：12 到 48 小時
    • 最低存儲持續時間：180 天

成本優化：根據數據訪問模式選擇合適的存儲類別可以顯著降低成本。使用生命週期策略自動在不同存儲類別之間轉換對象，或使用 S3 智能分層自動優化成本。

數據管理功能

  • 生命週期管理
    • 自動轉換存儲類別
    • 自動刪除過期對象
    • 基於前綴、標籤或對象大小配置規則
    • 可以應用於整個存儲桶或對象子集
  • 版本控制
    • 保留對象的多個版本
    • 防止意外刪除和覆蓋
    • 可以恢復刪除的對象
    • 可以與生命週期規則結合使用
  • 複製和複製
    • 跨區域複製 (CRR)：跨區域複製對象
    • 同區域複製 (SRR)：在同一區域內複製對象
    • 可以更改存儲類別和所有權
    • 可以與版本控制結合使用
  • 對象鎖定
    • 防止對象被刪除或覆蓋
    • 合規模式：不可覆蓋或刪除
    • 治理模式：特殊權限可以覆蓋保護
    • 法定保留：防止刪除，直到明確移除

安全性

  • 訪問控制
    • IAM 策略：基於身份的訪問控制
    • 存儲桶策略：基於資源的訪問控制
    • 訪問控制列表 (ACL)：基於對象級別的訪問控制
    • 預簽名 URL：臨時訪問
  • 加密
    • 服務器端加密 (SSE)
      • SSE-S3：使用 Amazon S3 管理的密鑰
      • SSE-KMS：使用 AWS KMS 管理的密鑰
      • SSE-C：使用客戶提供的密鑰
    • 客戶端加密：在上傳前加密數據
    • 默認加密：自動加密新對象
  • 網絡安全
    • VPC 端點：私有訪問 S3
    • 存儲桶策略限制 IP 和 VPC 端點
    • 阻止公共訪問設置

考試要點：了解不同存儲類別的特點和成本，以及如何使用生命週期策略優化成本。理解 S3 的安全機制，特別是不同加密選項和訪問控制方法。

性能優化

  • 請求速率
    • 每個前綴每秒 3,500 個 PUT/COPY/POST/DELETE 請求
    • 每個前綴每秒 5,500 個 GET/HEAD 請求
    • 通過使用隨機前綴或鍵名稱分區提高性能
  • 傳輸加速
    • 使用 CloudFront 的全球分布式邊緣節點
    • 加速上傳和下載
    • 適合跨地理位置傳輸大文件
  • 多部分上傳
    • 並行上傳對象的多個部分
    • 提高吞吐量和恢復能力
    • 推薦用於大於 100MB 的對象
  • S3 Select
    • 僅檢索對象的一部分
    • 減少傳輸的數據量
    • 提高查詢性能

Amazon EBS (彈性塊存儲)

卷類型

  • 通用型 SSD (gp2/gp3)
    • 平衡價格和性能
    • 適合啟動卷、開發和測試環境
    • gp3：獨立配置 IOPS 和吞吐量
    • gp2：性能與卷大小掛鉤
  • 預置 IOPS SSD (io1/io2)
    • 高性能、低延遲
    • 適合 I/O 密集型工作負載
    • io2 Block Express：最高性能
    • 支持多附加（僅限 io1/io2）
  • 吞吐量優化型 HDD (st1)
    • 低成本 HDD 卷
    • 適合頻繁訪問、吞吐量密集型工作負載
    • 不能用作啟動卷
  • 冷 HDD (sc1)
    • 最低成本 HDD 卷
    • 適合不頻繁訪問的工作負載
    • 不能用作啟動卷

選擇指南：根據工作負載的 IOPS、吞吐量和成本需求選擇合適的 EBS 卷類型。對於數據庫等 I/O 密集型應用，選擇 io1/io2；對於大數據處理等吞吐量密集型應用，選擇 st1；對於一般用途，選擇 gp2/gp3。

快照和備份

  • EBS 快照
    • 卷的時間點備份
    • 增量備份 - 只存儲變更的數據
    • 存儲在 S3 中（不可直接訪問）
    • 可以跨區域複製
  • 快照生命週期管理
    • 自動創建和刪除快照
    • 基於時間表或事件
    • 可以設置保留策略
  • 快照歸檔
    • 將快照移至歸檔層
    • 降低長期存儲成本
    • 恢復時間：24-72 小時
  • 快照鎖定
    • 防止意外或惡意刪除
    • 設置保留期
    • 合規模式或治理模式

高級功能

  • 多附加
    • 將單個 EBS 卷附加到多個 EC2 實例
    • 僅支持 io1/io2 卷
    • 所有實例必須在同一可用區
    • 需要集群文件系統
  • 快速快照恢復 (FSR)
    • 消除從快照恢復的卷的初始性能損失
    • 適合需要快速恢復的關鍵工作負載
    • 按快照和可用區收費
  • 彈性卷
    • 動態修改卷類型、大小和性能
    • 無需停止實例或分離卷
    • 只能增加卷大小，不能減小
  • 加密
    • 使用 AWS KMS 密鑰加密
    • 加密卷的快照自動加密
    • 可以設置默認加密
    • 最小性能影響

考試要點：了解不同 EBS 卷類型的性能特點和使用場景，以及如何使用快照進行備份和恢復。理解 EBS 加密和多附加等高級功能的應用場景。

Amazon EFS (彈性文件系統)

基本概念

  • 完全托管的彈性 NFS 文件系統
    • 自動擴展，無需預置容量
    • 支持 NFSv4.1 協議
    • 可以從多個 EC2 實例同時訪問
    • 跨可用區高可用性
  • 性能模式
    • 通用模式：默認，平衡延遲和吞吐量
    • 最大 I/O 模式：高吞吐量，略高延遲
  • 吞吐量模式
    • 突增吞吐量：基於存儲大小的吞吐量
    • 預置吞吐量：獨立於存儲大小的固定吞吐量
    • 彈性吞吐量：自動擴展吞吐量

存儲類別

  • EFS 標準
    • 默認存儲類別
    • 適合頻繁訪問的文件
    • 跨多個可用區存儲數據
  • EFS 不頻繁訪問 (IA)
    • 適合不經常訪問的文件
    • 成本比標準低約 85%
    • 有檢索費用
  • EFS 單區域
    • 數據存儲在單個可用區
    • 比標準存儲成本低約 47%
    • 適合不需要多可用區彈性的工作負載
  • EFS 單區域不頻繁訪問
    • 單可用區 + 不頻繁訪問
    • 最低成本 EFS 存儲類別
    • 適合不頻繁訪問且不需要多可用區彈性的數據

成本優化：使用生命週期管理自動將不經常訪問的文件轉移到 IA 存儲類別。如果不需要多可用區彈性，考慮使用單區域存儲類別降低成本。

安全性

  • 網絡訪問控制
    • VPC 安全組
    • NFS 級別訪問控制
    • IAM 身份驗證
  • 加密
    • 傳輸中加密：使用 TLS
    • 靜態加密：使用 KMS 密鑰
    • 創建時啟用，無法更改
  • 訪問點
    • 為應用程序提供特定目錄視圖
    • 強制執行用戶身份
    • 限制根目錄訪問

備份和恢復

  • AWS Backup 集成
    • 自動備份 EFS 文件系統
    • 設置備份計劃和保留策略
    • 跨區域備份
  • 時間點恢復
    • 恢復到特定時間點
    • 創建新的文件系統

考試要點：了解 EFS 與 EBS 的區別，特別是在多實例訪問和跨可用區可用性方面。理解不同存儲類別和性能模式的適用場景，以及如何優化成本和性能。

AWS Storage Gateway

類型和用例

  • S3 文件網關
    • 通過 NFS/SMB 協議訪問 S3
    • 本地緩存提供低延遲訪問
    • 適合文件共享、數據遷移
  • FSx 文件網關
    • 提供對 Amazon FSx for Windows File Server 的低延遲訪問
    • 本地緩存常用數據
    • 適合 Windows 文件共享
  • 卷網關
    • iSCSI 塊存儲接口
    • 數據存儲在 S3，作為 EBS 快照備份
    • 適合備份、災難恢復
  • 磁帶網關
    • 虛擬磁帶庫 (VTL) 接口
    • 與現有備份軟件兼容
    • 數據存儲在 S3 和 Glacier
    • 適合長期歸檔和替換物理磁帶

部署選項

  • 虛擬機設備
    • 在 VMware ESXi、Microsoft Hyper-V 或 KVM 上運行
    • 在本地數據中心部署
  • 硬件設備
    • 物理硬件設備
    • 預配置 Storage Gateway 軟件
    • 適合無法運行虛擬機的環境
  • Amazon EC2 實例
    • 在 AWS 雲中運行
    • 通過 Direct Connect 或 VPN 連接到本地環境

混合雲存儲：Storage Gateway 是構建混合雲存儲架構的關鍵組件，允許本地應用程序無縫訪問 AWS 存儲服務，同時保持低延遲訪問和與現有系統的兼容性。

AWS Snow 系列

設備類型

  • Snowcone
    • 最小的 Snow 設備
    • 8TB 存儲容量
    • 適合邊緣計算和數據傳輸
    • 可以通過 AWS DataSync 在線傳輸數據
  • Snowball Edge
    • 存儲優化：80TB 存儲容量
    • 計算優化：42TB 存儲容量，更強大的計算能力
    • 支持 EC2 兼容實例
    • 適合大規模數據遷移和邊緣計算
  • Snowmobile
    • 移動數據中心
    • 每個 Snowmobile 可傳輸高達 100PB 數據
    • 適合大規模數據中心遷移

用例

  • 大規模數據遷移
    • 將 PB 級數據遷移到 AWS
    • 適合網絡帶寬有限的情況
    • 比互聯網傳輸更快、更經濟
  • 邊緣計算
    • 在網絡連接有限的位置運行應用程序
    • 支持 EC2 兼容實例和 AWS Lambda
    • 可以在本地處理數據並僅發送結果
  • 災難恢復
    • 將數據備份到 AWS
    • 在災難恢復站點預先部署設備
  • 數據中心退役
    • 關閉數據中心時遷移數據
    • 適合一次性大規模遷移

考試要點：了解何時使用 Snow 系列設備而不是通過互聯網傳輸數據，以及如何選擇合適的 Snow 設備類型。理解 Storage Gateway 的不同類型及其適用場景。
